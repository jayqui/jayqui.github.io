Ruby Arrays & Hashes: Fun Facts

2015.5.17

Arrays are essentially ordered lists of data. In Ruby (along with many other languages), the items in the list can be any kind of object. And the lists are indexed by their place in the list, starting with 0 and going on until the end of the array. (In other words, they are integer-indexed".)

The official documentation on Ruby arrays contains a great deal of helpful information and examples, but I'll share a few helpful and overlooked ones before then contrasting Ruby's arrays with hashes. So without further ado, here are today's top 5 cool facts about Ruby arrays:

  1. While the typical array syntax is

    fruits = ["apple", "banana", "cherry", "dragonfruit", "elderberry", "fig"]

    arrays can also be created by splitting items via whitespace:

    fruits = %w(apple banana cherry dragonfruit elderberry fig)

    creates the very same array.
  2. Every beginner learns that Ruby array items can be accessed by the array name and the item position; for example, from the previous array,

    fruits[2] # => "cherry"

    . But it's less well known that several consecutive array can be accessed when a pair of arguments are put between the square brackets: fruits[2,3] = ["cherry", "dragonfruit", "elderberry"].
  3. Several items in a row can be created with the new method:

    three_greetings = Array.new(3, "hi") # => ["hi", "hi", "hi"]

  4. To remove duplicate items from an array, use the uniq! method (or, for a non-destructive method that preserves the original array and returns a copy of that array with duplicates eliminated, use uniq). Relatedly, if you have some nil values that you want to remove from your array, you should use the compact method.

    nihilism = [42, "meaning", "meaning", "of", "of", "life", "life", nil, nil, nil]

    nihilism.uniq # => [42, "meaning", "of", "life", nil]

    nihilism.compact # => [42, "meaning", "meaning", "of", "of", "life", "life"]

  5. Beginners often learn about the traditional methods of adding to and removing from the end (push and pop, respectively) and the front (unshift and shift, respectively) and the aesthetically pleasing shovel (<<) operator. But what they don't always learn is how to add and remove from the middle of an array. Here are the three relevant methods in action (assume these are each performed on the original fruits array):

    fruits.delete_at(2) # => "cherry", and "cherry" is removed from the array

    fruits.insert(1, "FEE", "FIE", "FOE", "FUM") # => ["apple", "FEE", "FIE", "FOE", "FUM", "banana", "cherry", "dragonfruit", "elderberry", "fig"]

    fruits.delete("banana") # => ["apple", "cherry", "dragonfruit", "elderberry", "fig"]
