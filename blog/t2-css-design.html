CSS Displays

What you need to know but may not have asked

2015.5.13

You may be familiar with HTML, which provides the structure for a web page, and CSS, which is the language for stylesheets that give formatting to the HTML outline.

But if you're like me, one of the hardest things to understand when first learning CSS is the display property. To understand it properly, we have to understand how display works on an HTML page.

First we have to understand the structure of an HTML page. Some elements are block level elements, meaning that they appear to start on a new line in the browser window. Examples inlcude the tags h1, p, ul, and li. Other elements are inline elements, meaning that they appear to continue on the same line as their neighboring elements. Examples include the tags a, b, em, and img. One of the chief uses of the div tag is to enclose content in block-level boxes so it can be formatted. The span element has a parallel function: it encloses inline content so that it can be formatted.

The most common CSS keywords that will be applied to the display property will be, unsurprisingly, inline and block. These will be used when we want to change the display property. For example, we might want to display an ordered list horizontally. li tags are normally block elements, but we can use CSS to change their display to inline. For a second example, we might want to display several images in a row, and to do this, we would set their display property to inline.

The display property can take various other keywords (full list here), but the most important one to mention here is inline-block. The inside of an element with inline-block formatting is formatted as a block-level box, but the element itself is formatted as an inline-level box. Basically, inline-block elements are like inline elements except they can have width and height. This can be a good way to make rows of boxes, for example. For example, check out this JSFiddle.
