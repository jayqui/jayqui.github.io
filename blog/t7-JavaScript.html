Here's a list of hints and things to remember for new programmers transitioning from Ruby to JavaScript.

What Ruby Has that JS Lacks

This is a pretty incomplete list, but I will probably continually add to it.

Ruby has but JS lacks example Workaround in JS example
string interpolation "#{variable1} and #{variable2} can just be written into a string" string concatenation variable1 + "and" + variable2 + "must be added to the string with the plus sign" (Note, this works even if the variables aren't srtrings because JS does automatic type conversions of these variables to strings. That wouldn't work with Ruby.)
negative indexing for arrays 
arr = [4,5,6]
arr[-1] # => -6
 arr[arr.length - 1]
arr.include? method arr = [4,5,6] && arr.include?("str") == false Use the detail that array.indexOf(something not in the array)equals -1 arr.indexOf("str")!= -1
arr.delete(thing) method if arr = [4,5,6,6,6,6] and arr.delete(6) is called then 6 is returned and arr becomes [4,5] Iterate through the array backwards (backwards to deal with shrinking array size) and use splice to delete the item each time it's found. Cf. here. 
for(var i = array.length - 1; i >= 0; i--) {
  if(array[i] === number) {
    array.splice(i, 1);
  }
}
This returns [6] and array becomes [4,5]

The Details of JavaScript

The first things that JS has or requires but Ruby does not are obvious:

JS has but Ruby lacks example Workaround in Ruby example
need for explicit return values
need for semicolons to end expressions
need for brackets to demarcate blocks brackets or do...end to demarcate blocks

And here are some slightly less obvious things:

JS has but Ruby lacks example Workaround in Ruby example
No arity: extra arguments to functions are just ignored in JS, but would raise an error with Ruby methods 
function fname(param1){
  // cool code here;
}

fname(3,4,5,6,7);
// just runs fname(3)
 splat operator 
def mname(*args)
  args.each do |arg|
    # do stuff to arg
  end
end

mname(3,4,5,6,7)
# sends the array [3,4,5,6,7] through #mname
automatic type conversions 'hi' + 5 // 'hi5' explicit type conversions 'hi' + 5.to_s # => 'hi5'
